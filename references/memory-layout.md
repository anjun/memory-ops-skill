# Memory Layout (OpenClaw)

## Canonical structure

- Long-term conclusions: `MEMORY.md`
- Daily timeline: `memory/YYYY-MM-DD.md`
- Project details: `memory/projects/*.md`
- Reusable patterns/checklists: `memory/patterns/*.md`

## Placement rules

### Put in MEMORY.md

- Stable preferences
- Durable decisions
- Rules that are reused often
- Index links to project docs

### Put in memory/projects/*.md

- Commands, logs, debugging sequences
- Failure analysis and root causes
- Tool-specific runbooks

### Put in daily log

- Time-ordered events
- One-off outcomes
- Temporary notes
